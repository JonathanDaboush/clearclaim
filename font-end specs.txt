================================================================================
                         CLEARCLAIM - FRONT-END SPECS
================================================================================

The system is a record & accountability platform - not a social tool.
It should feel like a digital case file builder: calm, factual, methodical.

Priority order: Trust > Clarity > Evidence visibility > Speed > Accessibility > Simplicity

--------------------------------------------------------------------------------
  TYPOGRAPHY
--------------------------------------------------------------------------------

  Fonts: Inter / Roboto / Source Sans (no decorative fonts)
  Body: 16px min  |  Metadata: 14px  |  Headings: 20-28px


--------------------------------------------------------------------------------
  NAVIGATION & LAYOUT
--------------------------------------------------------------------------------

  Three-layer architecture:

  Layer 1 - System Overview     Dashboard | Projects | Notifications | Account
  Layer 2 - Project Workspace   Timeline | Contracts | Evidence | Members | Settings
  Layer 3 - Record Detail       Evidence viewer | Contract viewer | Signature record

  Left sidebar navigation + breadcrumbs (e.g. Dashboard > Project > Evidence > Photo #14)
  Timeline is always the default project view.


--------------------------------------------------------------------------------
  SCREENS
--------------------------------------------------------------------------------

  1. AUTH
     Login -> TOTP -> Dashboard
     Fields: email, password, then TOTP code
     Backend: login, device verification, suspicious activity detection

  2. NEW DEVICE VERIFICATION
     Shows device info + location estimate -> Approve / Deny
     Backend: add device, log event, send notification

  3. IDENTITY VERIFICATION
     Optional - triggered for higher legal assurance
     Upload ID via provider -> status: pending / verified / rejected
     Sensitive docs never stored in system

  4. DASHBOARD
     Overview widgets: projects needing action, pending signatures,
     recent evidence, security alerts, billing notices
     Backend: projects, contracts, audit logs, notifications, security

  5. PROJECT LIST
     Columns: name, role, last activity, verification level
     Actions: create / open / leave project
     Backend: projects, memberships, roles

  6. PROJECT WORKSPACE
     Tabs: Timeline | Contracts | Evidence | Members | Settings
     Timeline is default tab

  7. TIMELINE
     Chronological event feed (contract created, revision, signature,
     evidence upload, dispute, role change, device change)
     Each card: user, timestamp, action, status -> click to expand
     Backend: audit logs, contracts, evidence, signatures

  8. CONTRACT LIST
     Columns: name, current version, status, last revision
     Actions: open / create contract

  9. CONTRACT VIEWER
     Sections: document viewer, version history, signatures
     Scroll, view history, view signing status

 10. CONTRACT REVISION
     Side-by-side diff (GitHub-style): green = added, red = removed
     Actions: approve / reject revision

 11. SIGNING
     Step 1: confirm review -> Step 2: enter TOTP -> Step 3: signature recorded
     Displays: timestamp, device, IP, contract hash
     Backend: ephemeral token -> verify TOTP -> hash contract -> store sig -> audit log -> notify

 12. EVIDENCE LIBRARY
     Grid of photos, videos, documents, notes
     Upload / search / filter (by date, user, type, contract, keyword)
     Evidence is immutable - no replacements, only additions

 13. EVIDENCE VIEWER
     Modal or fullscreen: zoom (scroll/pinch/double-click), pan, full-res
     Metadata: uploader, timestamp, hash, linked contract
     Keyboard shortcuts supported
     Documents: page nav, zoom, text selection

 14. EVIDENCE SHARING
     Options: email, secure link, export package
     Optional attachments: metadata, timeline context
     Packages: selected files + timeline excerpts + metadata records

 15. MEMBERS & ROLES
     Columns: user, role, verification status, device status
     Admin actions: change role, invite, remove

 16. DEVICE MANAGEMENT
     Columns: device name, location, last activity
     Actions: revoke / approve device

 17. SECURITY ALERTS
     Events: new device login, failed signing attempts, unusual location
     Actions: review, approve, lock account

 18. NOTIFICATIONS CENTER
     Types: contract revision, evidence change, security alert, project invite

 19. BILLING & SUBSCRIPTION
     Shows: tier, billing history, next payment, simple usage charts

 20. ACCESSIBILITY SETTINGS
     Options: text size, high contrast, reduced motion, captions, screen reader mode


--------------------------------------------------------------------------------
  INTERACTION PRINCIPLES
--------------------------------------------------------------------------------

  * Familiar patterns: uploads like Drive, logs like GitHub, feeds like Slack
  * Evidence capture in seconds: open project -> tap add -> photo/upload -> submit
  * Clear confirmations: "Evidence uploaded" / "Signature recorded" / "Revision saved"
  * Animations: minimal, subtle, optional (reduced motion support)
  * Data transparency: always show integrity signals (verified, timestamped, authenticated)


--------------------------------------------------------------------------------
  MOBILE
--------------------------------------------------------------------------------

  Optimized for: evidence capture, contract review, signing
  Floating "Add Evidence" button | large tap targets | swipe media navigation


--------------------------------------------------------------------------------
  ACCESSIBILITY
--------------------------------------------------------------------------------

  * High contrast mode, adjustable text, reduced motion, screen reader, keyboard nav
  * Status = color + text/icon (never color alone)
  * Image zoom + descriptions for visually impaired
  * Video captions + transcripts; notifications are visual + auditory
  * Full keyboard navigation for all core flows
  * Compliant with Accessible Canada Act & ADA


--------------------------------------------------------------------------------
  TYPICAL USER FLOW
--------------------------------------------------------------------------------

  Login -> Dashboard -> Open project -> View timeline -> Open contract
  -> Review revision -> Sign -> Upload evidence -> Share evidence

  Every step auto-appends to the audit log.

  The frontend feels simple; the backend handles cryptography,
  tamper prevention, audit chains, and security monitoring invisibly.
