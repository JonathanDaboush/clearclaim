1. User Accounts and Identity

Sign-up/Login/Password Management:
• Email + password only (no usernames)
• Forgot password and account removal supported
• Authenticator app (TOTP) required for login and signing

Identity Verification:
• Optional/required government ID verification for higher legal assurance
• Verification result stored (status, provider, timestamp), not raw sensitive IDs
• Guests and clients follow same verification protocols

Multi-Device Support:
• Trusted device system
• New devices require authenticator code + email verification
• Users can revoke devices at any time
• Lost/stolen devices cannot sign or tamper with logs

2. Projects, Groups, and Roles

Projects & Subgroups:
• All projects have a main party; subgroups can exist within projects
• Subgroup memberships and roles are managed independently from overall project membership
• Users can join/leave subgroups only with permission from existing members
• Soft deletion for leaving users; historical participation is preserved

Roles & Permissions:
• Role-based permissions (worker_manager, worker, legal_rep, client, guest)
  - worker_manager: full access + can invite/manage other workers
  - worker: create/revise contracts, add evidence, approve revisions, sign
  - legal_rep: approve revisions, add evidence, sign
  - client: sign contracts, view
  - guest: view only
• Actions such as signing, evidence addition/deletion, revision approval are restricted based on role
• Role changes can be made by user admin; easily managed via UI

Notifications:
• Role-based notifications for relevant updates

3. Contracts and Agreements

Signing Process:
• Users must explicitly confirm intent to sign
• Authenticator app code required for each signature
• Users may save signature images but cannot sign automatically with saved images
• Each signing event logged with user ID, timestamp, IP, device info, contract version, and hash

Revision Tracking:
• GitHub-style line-by-line comparison for contract changes
• All revisions must be approved by all parties
• Historical versions preserved for audit

Evidence Management:
• Evidence addition/deletion requires unanimous consent from all relevant parties
• All evidence is timestamped and hashed for integrity
• Users notified of changes or deletions

4. Security and Tamper-Prevention

Audit Logs:
• Append-only logs
• Cryptographically chained (hash of each log includes previous hash)
• Periodic snapshot hashes anchored externally or archived
• Includes event details, timestamps, device, and signing actions

Key Management:
• Hierarchical key system: root key (offline/HSM), system signing key (HSM), ephemeral session keys
• Ephemeral signing tokens bound to specific sessions/documents
• Key rotation supported; signed by previous keys
• Lost device recovery without breaking log integrity

Suspicious Activity Monitoring:
• New device logins, unusual activity, repeated failed signing attempts trigger alerts
• Alerts sent to users/admins
• Suspicious events temporarily restrict sensitive actions

5. Payment and Billing

Hybrid Pricing Model:
• Clients/businesses pay monthly subscription
• Guest accounts are free and dependent on project existence
• Employees/other participants can optionally be charged depending on subscription tier
• Users can view payment history and simple metrics explaining charges
• Apple-style “ease of understanding” visualization

6. User Experience and Apple-Style Simplicity

Signing/Contract Actions:
• Open contract → click sign → enter authenticator code
• Time to sign: ~10 seconds
• Users never see cryptography or key management details

Alerts & Guidance:
• Clear, short instructions for resolving issues (lost device, suspicious activity, recovery)
• Minimal steps for all security actions
• Device approval/revocation handled with a few clicks

Revision/Evidence Visibility:
• Users see chronological history of contract and evidence changes
• Color-coded diffs, added/deleted lines
• Timeline view similar to GitHub activity logs

7. Legal and Compliance Considerations

Identity Verification and Signatures:
• Complies with ESIGN, UETA (US), eIDAS (EU), PIPEDA (Canada)
• Strong evidence of identity, intent, and document integrity

Audit Trails & Tamper Evidence:
• Cryptographic chaining + HSM keys make logs tamper-evident

• Device revocation and multi-device authentication maintain audit validity
# 1. User Account Layer


	- **Account Database**
			- Stores email, password hash, authenticator status, device list, role, verification status
			- No sensitive identifiers like SIN stored
	- **Identity Verification Module**
			- Optional government ID check for high-assurance users
			- Stores verification status, timestamp, and provider reference
	- **Multi-Device Management**
			- Trusted devices stored per user
			- Device revocation possible anytime
			- New device login triggers email + authenticator confirmation

Evidence & Contracts:
	- User signs up → enters email/password → optional ID verification
	- Device added → user approves via authenticator + email
	- Any new device login → approval + logs event

• Historical versions preserved
# 2. Project & Group Layer

• All additions/deletions consented and logged
	- **Projects Table**
	- **Subgroups Table**
	- **Membership Table** (links user → project/subgroup → role)
	- **Roles Table** (defines permissions per role)


	- Admin creates project → defines main party and subgroups
	- Users invited → must be approved to join
	- Role-based permissions enforced:
			- worker_manager → full access + manage other workers
			- worker → create/revise contracts, add evidence, approve revisions, sign
			- legal_rep → review, approve revisions, add evidence, sign
			- client → sign contracts, view
			- guest → view only

# 3. Contract & Revision Layer


	- **Contracts Table**
	- **Contract Versions Table**
	- **Revision Changes Table** (diffs for line-by-line modifications)
	- **Evidence Table** (files, images, timestamps, hashes)

System balances maximum security with simplicity
	- Contract created → version 1 saved
	- Users make revisions → new version created → diff logged
	- All revisions require unanimous approval
	- Evidence added/deleted → all actions logged and consented


# 4. Signing & Authentication Layer

Legal evidence is strong, logs are tamper-proof, identity is verified
	- **Signing Module**
			- Authenticator app required for every signing action
			- Ephemeral signing token generated per document
	- **Signature Storage**
			- Cryptographic signature tied to user ID, contract version, timestamp
			- Optionally stored image for reference (cannot sign alone)


	- User clicks “Sign” → enters authenticator code
	- System creates signed log entry → hashes contract content
	- Log appended to audit chain

Users can sign from any device safely
# 5. Audit Log & Integrity Layer


	- **Append-only audit log table**
	- **Cryptographic hash chain** (each entry includes previous hash)
	- **Key Hierarchy:**
			- Root key (offline/HSM)
			- System signing key (HSM)
			- Ephemeral session keys per device
	- **Periodic snapshot hashes** (archived externally)

Guests/clients follow the same strict security protocols
	- Every action → logged → hashed → signed by system key
	- Verification can detect tampering
	- Lost devices do not affect log integrity


# 6. Security & Monitoring Layer

System is designed to scale, accommodate device loss, and maintain cryptographic integrity
	- **Suspicious Activity Detector**
			- Monitors login attempts, device changes, unusual signing behavior
	- **Alerts Module**
			- Sends emails or in-app notifications
	- **Recovery Module**
			- Supports new device setup with identity verification
			- Revokes lost device access


	- Suspicious login or activity detected → temporary restrictions
	- Alerts sent to user/admin
	- Recovery and new device approval possible without breaking logs

Apple-style design principle applied: complexity handled internally, minimal user friction externally
# 7. Payment & Subscription Layer


	- **Subscription Table**
	- **Payment History Table**
	- **Metrics Table** (explains charges in simple UI)
	- **Hybrid Pricing Model**
			- Clients/businesses pay monthly
			- Guest accounts free
			- Employee/participant charges optional

# Functions Diagram (Outline)

  User Account Layer
    - Sign-up(email, password)
    - Login(email, password, TOTP)
    - Forgot password(email)
    - Remove account(user_id)
    - Add device(user_id, device_info, TOTP, email verification)
    - Revoke device(user_id, device_id)
    - Identity verification(user_id, provider, status)

  Project & Group Layer
    - Create project(name, main_party)
    - Create subgroup(project_id, name)
    - Invite user(project_id, subgroup_id, user_id, role)
    - Change role(user_id, project_id, subgroup_id, role)
    - Leave project/subgroup(user_id, project_id, subgroup_id)

  Contract & Revision Layer
    - Create contract(project_id, content)
    - Revise contract(contract_id, new_content)
    - Approve revision(contract_version_id, user_id)
    - Add evidence(contract_id, file_url, hash)
    - Delete evidence(evidence_id, unanimous consent)

  Signing & Authentication Layer
    - Sign contract(contract_version_id, user_id, TOTP)
    - Store signature(signature_data, image_url)
    - Generate ephemeral signing token(document/session)

  Audit Log & Integrity Layer
    - Log event(event_type, user_id, device_id, contract_id, contract_version_id, details)
    - Chain log(prev_hash, hash)
    - Snapshot log(hash, archive)

  Security & Monitoring Layer
    - Detect suspicious activity(user_id, event)
    - Send alert(user_id, event_type, content)
    - Restrict actions(user_id, event_type)
    - Recover device(user_id, identity verification)

  Payment & Subscription Layer
    - Record payment(user_id, amount, method)
    - Update subscription(user_id, tier, status)
    - Generate metrics(user_id)

  User Interface Layer
    - Sign contract (UI: click, enter TOTP)
    - View timeline (UI: chronological, color-coded diffs)
    - Manage devices (UI: approve/revoke)
    - View evidence/revisions (UI: timeline, diff)
    - Receive alerts (UI: notifications, instructions)

# 10. Legal Compliance Additions

Accessibility:
- Screen reader support, keyboard navigation, high-contrast mode.
- Accessibility statement published and contact info for accessibility issues.

Privacy Policy & Terms of Service:
- Clear privacy policy and terms of service displayed to users.
- Includes user rights, data handling, dispute resolution, and governing law.

Data Breach Notification:
- Process for notifying users and authorities in case of a data breach.

Data Localization & Cross-Border Transfers:
- If data is stored outside Canada, users are informed and PIPEDA compliance ensured.

Automated Record Retention/Deletion:
- Records are retained as required by law and industry standards.
- Users can delete accounts, but audit logs and evidence remain immutable for legal and compliance reasons.

Children’s Privacy (COPPA):
- If users under 13 are allowed, parental consent and special protections are implemented.

Right to be Forgotten:
- Users may request deletion of personal data, but audit logs and evidence are not deleted to preserve legal integrity.

User Consent Management:
- Users can opt-in/opt-out of marketing and non-essential data collection.
- Consent for data processing is recorded and managed.

Dispute Resolution & Jurisdiction:
- Governing law and dispute resolution process specified in terms of service.
